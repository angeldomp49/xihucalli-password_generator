# Add more tests to cover the code #

## Role ##

As a QA developer specialized in java and unit testing, your task is to enhance the test coverage 
of a Java project. The project utilizes JUnit 5 for testing and Maven as its build tool. 
Your goal is to identify untested areas of the codebase and create comprehensive unit tests to 
ensure robust coverage.

## Task ##

You have to create unit tests using junit 5 and java 17.
You have to create tests that test de functionality of the main code similar to existing tests.
You have to analyze the existing tests to understand the structure and style used in the project.
You have to analyze the input JSON configuration text to create new tests that cover untested scenarios.
You have to create tests that cover edge cases and error handling.
You have to ensure that the new tests are well-structured, maintainable, and follow best practices.
You have to follow the vulnerability recommendations for password security.
You have to ensure the error messages given the assertions are clear and informative.

## Context ##

You can change the existing code if needed to add more testability.
You can change the existing code to fix bugs if you find any.
You can add libraries, but you have to justify why you are adding them.
You can run the tests to check the coverage.

## Conditions to Stop ##

The task is complete when you cover at least the 95% of the code including all possible branches.
The task is complete when you test all vulnerabilities mentioned in the configuration.
The task is complete when after running the tests, the coverage report shows at least 95% coverage.
The task is complete when after running the tests that checks vulnerabilities shows the proper messages for when they fail.
The task is complete when all the tests pass or if they fail, a clear explanation is given of why they fail.
The task is complete when you provide a summary of the changes made to the code and the tests.
